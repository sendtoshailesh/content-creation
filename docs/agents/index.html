Orchestrator

all phases

The single entry point that coordinates all specialist agents in sequence, reads and updates pipeline status between every phase, and enforces quality gates.

⚙
content-pipeline @content-pipeline
Orchestrates the full visual-first content strategy pipeline. Coordinates specialist agents from clarifying questions through mandatory visual planning, blog, visuals, social posts, and video script. Reads and updates content/pipeline-config.md between every phase. Enforces the quality gate sequence so no phase advances without the prior gate passing.

optional

Chrome reading list idea discovery
apple-notes-curator @apple-notes-curator
Generates content ideas from Apple Notes. Reads your notes, auto-categorizes by subject area, clusters related notes into themes, scores by priority, and queues ideas interactively. Use to discover content ideas from notes you've captured on the go.
Apple Notes idea discovery

LinkedIn / X / Reddit GitHub Stars idea discovery

Phase 0 — Reference & Research

phase 0

Ground the content in current market reality and authoritative sources before any writing begins. Outputs: content/reference-brief.md and content/trend-research.md.

reference-discovery @reference-discovery
Web reference discovery agent inspired by NotebookLM. Searches the web for relevant references across 6 content categories, presents results for interactive user curation (select/reject), and writes accepted references into pipeline-config.md. Use before reference analysis to populate the pipeline with high-quality sources.
web search source curation
trend-researcher @trend-researcher
Market intelligence analyst for pre-writing research. Identifies emerging trends, performs competitive analysis, and provides data-backed insights to strengthen content with real numbers, benchmarks, and market context. Use before writing to ground content in current market reality.
market intelligence benchmarks competitive analysis

Phase 1 — Planning

phase 1

Turn a topic into a research-backed creative brief, STORM content plan, scope decision (single post vs. series), dimension analysis, and a visual opportunity map. Ends with explicit user confirmation before any content is written.

content-strategist @content-strategist
Use for content planning, audience research, and outlining. Conducts 8–12 clarifying questions, defines audience/tone/channels, and produces a structured content strategy with distribution-aware outline.
strategy outline creative brief
content-researcher @content-researcher
STORM-inspired content research agent. Runs the four-phase content-research pre-stage (perspective discovery, contradiction map, ranked synthesis, self peer-review, outline tree) and writes content/content-research-map.md before the strategy outline is written. The content-track twin of visual-strategist.
STORM research contradiction map outline tree
visual-strategist @visual-strategist
Mandatory visual strategy agent. Creates a visual opportunity map from strategy docs or blog drafts and plans blog, LinkedIn, Medium, Substack, and LinkedIn Article visual assets before rendering.
visual planning opportunity map mandatory
infographic-art-director @infographic-art-director
Infographic creative director for visual-first content. Researches design references, chooses infographic types, writes visual design briefs, and enforces infographic-first quality before rendering.
infographic briefs art direction P0/P1 visuals

Phase 2 — Content Creation

phase 2

Write the long-form blog post and render all visual assets. Every section exceeding 400 words without an image reference gets a [VISUAL: ...] marker. All rendered visuals pass the REVR hard gate (score ≥ 85) before advancing.

blog-writer @blog-writer
Use for writing long-form technical blog posts. Takes a strategy outline and produces publication-ready Markdown with concrete data, case studies, and visual integrations.
long-form blog ~3,000 words case studies
visual-renderer @visual-renderer
Use for generating visual assets — PNG charts via matplotlib, SVG graphics, and Mermaid diagrams. Follows the shared design token system for consistent visuals across all content.
PNG / SVG / Mermaid design tokens 320 DPI
image-content-agent @image-content-agent
Use for hero/backdrop/illustrative imagery. Defaults to a deterministic, free, offline programmatic backdrop; optionally calls an external image model for photoreal looks. Grounds on the creative brief + brand tokens, runs a deterministic QA pre-screen, and hands off to visual-reviewer. Does NOT make diagrams/infographics/exhibits — those stay in visual-renderer.
hero imagery programmatic (default) AI opt-in
visual-reviewer @visual-reviewer
Visual QA critic agent. Reviews rendered visual assets (PNG, SVG, Mermaid) for layout defects, readability, design token compliance, and reader comprehension. Produces a findings report for visual-renderer to fix. Uses GitHub Copilot rubber-duck review; no model-family switch required.
visual QA REVR gate score ≥ 85 required

Phase 3 — Quality & SEO

phase 3

Multi-tier quality cascade: rubber-duck content review, live source fact-checking, and SEO optimization. All reviewers emit the same severity-graded findings schema (Error / Warning / Info) defined in compliance-severity.md.

quality-reviewer @quality-reviewer
Use for reviewing and improving content quality. Audits blog posts, social posts, and visuals against quality standards. Fixes vague claims, missing data, broken visuals, and tone issues.
content QA Error/Warning/Info auto-fix
grounded-content-reviewer @grounded-content-reviewer
Validates created content against real-world sources via web search. Grounds blog posts, social posts, and video scripts by verifying data points, technical claims, examples, and scenarios against authoritative web sources. Fixes gaps and inaccuracies in-place.
fact-check live web search source freshness
seo-optimizer @seo-optimizer
SEO optimization specialist for blog content. Analyzes keyword opportunities, optimizes headings and structure, writes meta descriptions, and ensures content is search-engine friendly while remaining reader-first. Use after blog writing to optimize for organic traffic.
SEO meta descriptions keyword optimization
brand-guardian @brand-guardian
Brand consistency guardian for content pipelines. Audits all content pieces — blog, social posts, visuals, scripts — to ensure consistent voice, tone, design tokens, and messaging. Use after content creation to catch inconsistencies before publishing.
brand audit voice/tone design tokens

Phase 4–6 — Distribution

phases 4–6

LinkedIn is always generated first. Additional channels (X/Twitter, Reddit, Reel/Short, YouTube, deck, Medium/Substack) are user-selected after LinkedIn. Each social agent uses Unicode bold/italic for native LinkedIn/X formatting; Reddit uses standard Markdown only.

social-strategist @social-strategist
Cross-platform social media strategist. Develops unified social distribution strategy before individual platform agents run. Defines messaging themes, posting sequence, timing, and cross-promotion plan. Use before running individual social agents.
distribution strategy cross-platform plan
social-linkedin @social-linkedin
Use for creating LinkedIn posts from blog content. Produces both plain-text and Unicode-formatted versions optimized for LinkedIn's algorithm and audience. When a visual pack exists, produces visual-first posts with carousel/exhibit references in Practitioner or Executive persona mode.
LinkedIn Unicode bold always generated
social-twitter @social-twitter
Use for creating an X/Twitter single tweet from blog content. Produces one standalone tweet (≤280 chars) with a single visual attachment reference. When a visual pack exists, references the most relevant x-card or x-exhibit image.
X / Twitter ≤ 280 chars user-selected
social-reddit @social-reddit
Use for creating Reddit posts from blog content. Writes technical, discussion-oriented posts with standard Markdown for subreddits like r/MachineLearning, r/ExperiencedDevs, r/artificial.
Reddit standard Markdown user-selected
reel-video @reel-video
Use for creating short-form video (reel) scripts from blog content. Produces 60–90 second scripts with screen recording cues and voiceover narration for Instagram Reels, YouTube Shorts, and LinkedIn video.
short video 60–90 seconds user-selected
deck-builder @deck-builder
Build an optional slide deck (PPTX + PDF) from finalized blog + LinkedIn content, with per-slide speaker notes in a humor + intellectual voice, organized by topic. Runs only after the blog and LinkedIn post are finalized; exports only after the user finalizes the deck. Optional — never a publishing gate.
PPTX + PDF Marp source optional
video-scriptwriter @video-scriptwriter
Use for creating YouTube video scripts from blog content. Produces timed scripts with visual cues, slide maps to existing PNGs, thumbnail concepts, and YouTube descriptions.
YouTube script timed cues user-selected
content-repurposer @content-repurposer
Content repurposer that transforms a published blog post into additional formats — email newsletters, slide deck outlines, podcast talking points, one-page summaries, and infographic briefs. Use after the blog is published to maximize distribution reach.
newsletter podcast one-pager
platform-distiller @platform-distiller
Generates one unified excerpt for Medium, Substack, and LinkedIn Article from a completed blog post. User copies the same content to all three platforms. All output points to the GitHub Pages canonical URL. Use after web-publisher and social-publisher.
Medium / Substack LinkedIn Article step 12

Phase 8 — Measurement

optional · time-delayed

Closes the content-as-experiment loop. The creative brief declares a falsifiable hypothesis (§4b); post-publish-review scores actual engagement against it and harvests the outcome back into the idea queue. Never a publishing gate.

post-publish-review post-publish-review skill
Close the content-as-experiment loop. After a piece is published, compare its actual engagement against the falsifiable Content hypothesis from the creative brief, render a go/no-go verdict (validated / partially validated / invalidated / inconclusive), and write the outcome to the hypothesis ledger.
hypothesis ledger ~48h / ~7 day window optional
post-run-reflection post-run-reflection skill
Close the discovery loop. After a content run finishes, reflect on what the run itself surfaced — cut scope, open questions, unwritten series parts, reader-perspective gaps, and validated angles from the hypothesis ledger — and harvest them into the idea queue as ranked, ready-to-pick follow-ups.
idea harvesting idea-queue.md optional

Skills

cross-cutting

Reusable procedures invoked by agents throughout the pipeline. Skills are never gates; they are tools. Each writes a well-defined output artifact.

Planning & Tracking

creative-brief skill
Turn a topic and clarifying-question answers into a structured Creative Brief — including a mandatory falsifiable Content hypothesis — that grounds every downstream agent.
content-scope-assessment skill
Assess whether a topic is too comprehensive for a single post and recommend a multi-part series. Scores 8 signals (0–16); enforces per-part 2,400-word floor before recommending a split.
multi-dimensional-analysis skill
Analyze a topic across three dimension types — persona, best practice, and Azure Well-Architected Framework pillars — to inform series planning, content depth, and social distribution angles.
content-decision-record skill
Record consequential, hard-to-reverse content decisions ADR-style with the alternatives that were rejected and why. Writes to content/content-decision-log.md. Optional, lightweight, never a gate.
run-tracking skill
Give a content run a durable scratchpad that survives context loss and series gaps. Maintains a per-run phase log and a compaction handoff under content/_tracking/<run-slug>/.

Research & Sources

content-research skill
STORM-inspired pre-stage that builds a contradiction-aware, self-reviewed content plan. Discovers reader-perspectives, maps where sources and personas disagree, ranks thesis and key arguments by confidence, self-critiques the plan for bias.
reference-analysis skill
Fetch and synthesize online reference URLs for content creation. Analyzes pipeline-config reference URLs and writes content/reference-brief.md before writing begins.
source-grounding skill
Rank and cite sources by relevance and authority — not by vendor. All publishers (Microsoft, GitHub, independent vendors, analysts, researchers, community) are treated equally.

Visual Pipeline

visual-content-planning skill
Create a mandatory visual opportunity map from a strategy document or blog draft. Use before writing or rendering visuals to plan diagrams, infographics, comic/storyboard explainers, LinkedIn card packs, and executive exhibits.
visual-research skill
STORM-inspired four-phase visual planning pre-stage. Discovers reader-perspectives, maps conflicts, ranks visual decisions by confidence, self-critiques the plan, and writes a hierarchical mind map to content/visual-style-map.md.
visual-style-router skill
Deterministically pick a visual STYLE/MEDIUM (data-exhibit, typographic, hand-drawn, blueprint, editorial-illustration, diagram-as-code) per asset. Enforces package-level style diversity so no two adjacent visuals share a look.
style-rendering skill
Drive the six style adapters (data-exhibit, typographic, hand-drawn via Rough.js/xkcd, blueprint, editorial-illustration, diagram-as-code via D2/Mermaid/Graphviz) with shared design tokens and the existing inspector/reviewer gates.
visual-rendering skill
Generate visual assets for content — PNGs via matplotlib, SVGs via Python, and Mermaid diagrams. Follows the shared design token system for consistent visuals across all content.
visual-pack-generator skill
Generate platform-optimized visual asset packs from blog content for visual-first distilled posts. Supports Practitioner (carousel) and Executive (exhibit) persona modes.
visual-reverse-review skill
Reverse-Engineering Visual Review (REVR) — hard quality gate. Reads meaning back out of a rendered visual blind (pixels only), back-translates against the source concept, scores the semantic gap on a 0–100 rubric. Score ≥ 85 + zero legend gaps required to pass.
infographic-design-system skill
Create research-backed infographic design briefs before rendering visual assets. Use for infographics, one-pagers, comic/storyboards, social card packs, executive exhibits, and visual-first blog/social content.
vision-grounding skill
Build grounded, consolidated prompts for AI image generation. Describe reference images with a vision model and assemble a single prompt from subject + creative brief + brand guidelines + constraints.

Quality Review

critic-review skill
Tier 1 rubric critic and reference-grounded fact-check desk. Runs a G-Eval-style constitution judge and a faithfulness check anchored to the reference brief. Emits compliance-severity findings with confidence/risk fields. Auto-approves clean low-risk majority; abstains to human on high-stakes claims.
visual-review skill
Review rendered visual assets (PNG, SVG, Mermaid diagrams) for layout defects, text overflow, readability, design token compliance, data accuracy, and reader comprehension. Produces a structured findings report for the visual-renderer agent to act on. Mandatory pipeline step before publishing (Phase 2c).

Distribution & Formatting

practitioner-projects skill
Turn a blog post's core concept into a hands-on "Build it yourself" section of three structured, repo-grounded projects (beginner → intermediate → advanced). Each project presents interchangeable tools as equals regardless of vendor.
deck-builder skill
Build an optional presentation deck from finalized blog + LinkedIn content. Generates a Marp Markdown deck with per-slide speaker notes written in a humor + intellectual voice. Exports PPTX and PDF after user finalizes.
linkedin-carousel skill
Build one consolidated LinkedIn document carousel (PDF) that stitches a blog's publish-ready visuals into a single swipeable story — cover + one captioned slide per visual + CTA.
unicode-formatting skill
Format text with Unicode Mathematical Bold and Italic characters for LinkedIn and X/Twitter posts. Use when creating social media content that needs native bold/italic rendering without Markdown support.

Post-Publish & Discovery

post-publish-review skill
Close the content-as-experiment loop. Compare actual engagement against the brief's §4b hypothesis after the measurement window, render a verdict, and write the outcome to the hypothesis ledger. Optional, time-delayed, never a gate.
post-run-reflection skill
After a content run finishes, reflect on what was surfaced — cut scope, open questions, unwritten series parts, perspective gaps — and harvest ranked follow-up ideas into the idea queue. Optional, never a gate.
feed-curation skill
Read configured blog rolls, newsletters, and RSS feeds, classify articles by subject area, extract key insights, synthesize content ideas, and rank them for the idea queue.
