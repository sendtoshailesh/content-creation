#### [MAJOR-01] Long-Form Platform Distribution section contains stale "No images or media" statement

**File**: `content/pipeline-config.md`, lines 199–201
**Finding**: The `### Long-Form Platform Distribution (Step 12)` section still reads:
> `Text-only summaries generated by platform-distiller. Each drives traffic back to the GitHub Pages canonical URL. **No images or media — copy-paste ready.**`

Phase 2 of this implementation explicitly reversed the image prohibition in `platform-distiller.agent.md` (changes log, Phase 2 entry: "Replaced `Critical Constraint: Text-Only Output` section (image prohibition) with `Visual-First Output Policy`"). Phase 3 modified `pipeline-config.md` and added the `## Distillation Settings` section and the `## Pipeline Steps Reference` table, but left this stale description untouched.

**Impact**: Pipeline users reading this section will conclude that platform-distiller still outputs text-only content — contradicting the Distillation Settings section (which references carousel slides and visual assets) and the updated Publish Sequence (which describes uploading hero images and exhibits to Medium/Substack/LinkedIn Article). This is a direct factual contradiction within the same file.

**Evidence**:
- Stale statement: `pipeline-config.md` line 201
- Phase 2 reversal: changes log lines 56–57; `.github/agents/platform-distiller.agent.md` (Visual-First Output Policy)
- Contradicting new content: `pipeline-config.md` lines 245–246 (Distillation Settings), lines 322–327 (Publish Sequence visual entries)

**Recommendation**: Update `### Long-Form Platform Distribution (Step 12)` description to: `Visual-first summaries generated by platform-distiller when a visual pack is present; text-only fallback when no pack exists. Each output drives traffic back to the GitHub Pages canonical URL.`

#### [MAJOR-02] Step 12 description in Pipeline Steps Reference table uses stale "text-only excerpts" label

- New table created in Phase 3: changes log line 69

**Recommendation**: Update Step 12 description to: `Generate Medium / Substack / LinkedIn Article — visual-first when visual pack exists; text-only fallback otherwise.`

#### [MINOR-01] Default value for `distillation_persona_mode` deviates from spec

**File**: `content/pipeline-config.md`, line 245
**Finding**: The Details spec (line 434) specifies the default value as `ask`:
> `| **Persona mode** | ask |`

The Details spec rationale (line 439) explicitly states: "Default is `ask` — prompts the user to select persona mode at the start of each distillation run. This allows running both modes sequentially for the same content."

The actual implementation sets the default to `practitioner`:
> `| distillation_persona_mode | practitioner | ...`

The changes log records the value as `practitioner` (line 59) without explaining the deviation.

**Impact**: Operators expecting the pipeline to prompt for persona mode selection on each run will instead get Practitioner mode silently applied. This is a behavioral difference — not just cosmetic. Users running the pipeline without reading the config may unknowingly generate Practitioner-only visual packs when they wanted Executive mode.

**Severity**: Minor rather than Major because `practitioner` is a valid value, the field is documented as editable, and the Description column explicitly lists all three valid values including `ask`. The functional capability is preserved; only the default behavior changed without documented rationale.

**Evidence**:
- Details spec default: `.copilot-tracking/details/2026-05-13/visual-first-distillation-system-details.md` line 434
- Actual value: `pipeline-config.md` line 245
- Changes log: line 59 (records `practitioner` without deviation note)
