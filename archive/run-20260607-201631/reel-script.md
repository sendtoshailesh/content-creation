## Screen Recording Notes

- **App**: VS Code with GitHub Copilot extension active
- **Theme**: Dark theme (better contrast for vertical video)
- **Settings to show**:
  - Copilot Chat panel open (sidebar)
  - Model picker dropdown visible (top of chat panel)
  - A simple code file open (Python or TypeScript)
- **Actions to perform**:
  1. (0:08) Open GitHub Docs model multipliers page in browser, scroll to the table
  2. (0:22) Open VS Code Settings (Cmd+,), search for "copilot model", change to GPT-4.1
  3. (0:30) Open Copilot Chat, type: "rename this variable to camelCase" on a simple function, show GPT-4.1 responding correctly
  4. (0:45) Open model picker dropdown, click GPT-4.1 to set as default
- **Zoom**: Use 150-200% zoom on the relevant UI area. Avoid showing the full IDE — crop to the panel/dialog being discussed.

## Voiceover Script (Full)

> Your AI code assistant has models that cost a hundred and twenty times more than others. For the exact same prompt.
>
> GitHub Copilot's cheapest model is zero-point-two-five-x. Its most expensive is thirty-x. Starting June first, you pay per token.
>
> That means one Opus fast-mode request costs the same as a hundred and twenty nano requests.
>
> Here's the fix. Sixty to seventy percent of your coding tasks are simple. Variable renames. Docstrings. Import fixes. The cheapest models handle these identically.
>
> Apple's research confirmed it. Expensive models burn extra tokens on simple tasks with zero quality gain.
>
> A team I worked with applied this. Three thousand a day dropped to nine seventy. That's seven hundred forty K a year.
>
> Your move. Open VS Code. Switch your default to the cheapest model. Right now that's GPT-4.1.
>
> That's Part 1. Part 2 covers caching — up to ninety percent savings. Follow for that.
