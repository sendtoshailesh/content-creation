The chart above shows every GitHub Copilot model and its billing multiplier for June 2026. The bar your team defaults to is probably not the one you’d choose if you saw this chart first.

Starting June 1, 2026, GitHub Copilot moves from flat-rate to consumption billing. The same developer prompt now costs 0.25x to 30x depending on which model answers it — a 120x spread your budget didn’t account for.

The research:

▸ RouteLLM (LMSYS, 2024): 95% task quality achievable at 75% lower model cost
▸ CascadeFlow: 69% spend reduction with no code quality regression
▸ Apple ML Research (2025): reasoning models waste tokens on simple tasks — no quality improvement
▸ GPT-4.1 / GPT-5 mini: $0 on existing paid plans — available today, rarely default
▸ Claude Opus 4.6: 120x the cost of GPT-5 mini — your devs’ “just ask the best model” habit

The governance decision:

1. Set default model policy at the org level (not per-dev choice) — most tasks need Tier 1
2. Reserve Tier 3 (20-30x) for architecture, security reviews, and cross-system design
3. Instrument usage now — visibility before June 1 billing kicks in
4. Establish spend guardrails: quota alerts, team-level budget caps
5. Define quality checkpoints — not all code review needs Claude Opus

Case study: 100-developer team under this framework.
Pre-policy: $3,000/day in model spend
Post-policy: $970/day
Annualised saving: $740,000 — 68% reduction, zero quality regression reported

This is a model governance question, not a developer tooling question. The teams not auditing their default model choices before June 1 will discover the spread on their first consumption invoice.
